# 目录结构设计-前端

在上一章中我们介绍了如何使用[Tumo-AntV](https://github.com/Tumo-Team/Tumo-AntV) 项目，我之前也提到了这个项目遵循`Vue-Element-admin`项目的设计，其实很多代码逻辑都是保证原始项目中的（因为区别仅仅在于UI框架的不同）。

首先，让我们先看下项目的目录结构设计：

```
├── build				-- 项目构建目标目录
├── mock 				-- Mock.js配置目录（后期需要移除）
├── public				-- 不被路由拦截的资源目录
├── src
│   ├── App.vue
│   ├── api				-- 后端接口配置目录
│   ├── assets			-- 静态资源目录
│   ├── components		-- 公共组件目录
│   ├── directive		-- Vue指令封装
│   ├── filters			-- 过滤器
│   ├── icons
│   ├── layout
│   ├── main.js
│   ├── permission.js
│   ├── router
│   ├── settings.js
│   ├── store
│   ├── styles
│   ├── utils
│   └── views
└── vue.config.js
├── jest.config.js
├── jsconfig.json
├── LICENSE
├── README.md
├── babel.config.js
```
